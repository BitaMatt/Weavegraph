# 心灵导图可视化交互系统 - 产品需求文档

## 概述
- **摘要**: 一个支持可视化交互的心灵导图系统，专为销售类人士设计，用于制作和管理潜在客户网，提供直观的用户界面和丰富的交互功能。
- **目的**: 帮助销售人士更有效地管理潜在客户关系网络，提高销售效率和客户转化率。
- **目标用户**: 销售类人士，包括销售代表、销售经理、客户关系管理人员等。

## 目标
- 快速文本输入生成思维导图元素
- 支持点击空白处新增元素（使用上次生成的模板）
- 提供元素间的连接线功能，支持连接线命名
- 实现查看模式，鼠标悬停时放大显示详细信息并重点显示相关关系网
- 提供其他方便制作和查看导图的功能

## 非目标（范围外）
- 不需要实现复杂的自动布局算法
- 不需要支持多人协作编辑
- 不需要支持导出为特定格式（如PDF、Word等）
- 不需要支持云存储同步

## 背景与上下文
- 思维导图是一种有效的思维工具，帮助用户组织和可视化想法
- 现有的思维导图工具要么过于复杂，要么交互不够直观
- 本系统将专注于提供简单直观的可视化交互体验

## 功能需求
- **FR-1**: 快速文本输入生成思维导图元素
- **FR-2**: 点击空白处新增元素，使用上次生成的模板
- **FR-3**: 点击元素边框生成连接线头，支持连接到其他元素并命名连接线
- **FR-4**: 查看模式，鼠标移动到元素上时放大显示详细信息
- **FR-5**: 查看模式，鼠标移动到元素上时重点显示相关的关系网涉及的元素
- **FR-6**: 支持元素的编辑和删除
- **FR-7**: 支持基本的拖拽和调整元素位置

## 非功能需求
- **NFR-1**: 响应速度快，操作流畅
- **NFR-2**: 界面美观，符合现代设计趋势
- **NFR-3**: 支持Windows操作系统
- **NFR-4**: 数据本地存储，使用exe所在根目录的存储文件夹

## 约束
- **技术**: 使用Electron框架开发，使用前端技术实现，可使用开源库
- **业务**: 无特殊业务约束
- **依赖**: 依赖前端可视化库（如D3.js、Cytoscape.js等）
- **环境**: 支持本地运行，无需网络环境
- **版本管理**: 使用GitHub进行版本管理

## 假设
- 用户具备基本的计算机操作能力
- 用户使用现代浏览器访问系统
- 系统仅在本地运行，不需要服务器支持

## 验收标准

### AC-1: 快速文本输入生成
- **Given**: 用户打开系统
- **When**: 用户输入文本并确认
- **Then**: 系统生成对应的思维导图元素
- **Verification**: `human-judgment`
- **Notes**: 输入应该响应迅速，元素生成位置合理

### AC-2: 点击空白处新增元素
- **Given**: 用户已生成至少一个元素
- **When**: 用户点击空白处
- **Then**: 系统在点击位置生成新元素，使用上次生成的元素作为模板
- **Verification**: `human-judgment`
- **Notes**: 新元素应该继承上次元素的样式和属性

### AC-3: 元素连接功能
- **Given**: 系统中存在至少两个元素
- **When**: 用户点击第一个元素的边框，然后点击第二个元素
- **Then**: 系统在两个元素之间创建连接线，并允许用户为连接线命名
- **Verification**: `human-judgment`
- **Notes**: 连接线应该清晰可见，命名输入框应该易于使用

### AC-4: 查看模式功能
- **Given**: 系统中存在多个元素和连接
- **When**: 用户鼠标移动到某个元素上
- **Then**: 系统放大显示该元素的详细信息，并重点显示与该元素相关的其他元素
- **Verification**: `human-judgment`
- **Notes**: 放大效果应该平滑，相关元素的高亮应该明显

### AC-5: 元素编辑和删除
- **Given**: 系统中存在元素
- **When**: 用户选择元素并执行编辑或删除操作
- **Then**: 系统相应地更新或删除元素
- **Verification**: `human-judgment`
- **Notes**: 编辑操作应该直观，删除操作应该有确认提示

### AC-6: 元素拖拽和调整
- **Given**: 系统中存在元素
- **When**: 用户拖拽元素
- **Then**: 元素跟随鼠标移动，连接线自动调整
- **Verification**: `human-judgment`
- **Notes**: 拖拽操作应该流畅，连接线调整应该自然
